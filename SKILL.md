# Claim Evidence Finder

给任何观点、论断、假设找到权威学术证据。

> **一句话**：你说一句话，它自动找出学术界关于这句话的所有证据。

---

## 快速上手（零配置）

> 🚀 不需要申请任何 API，直接能用。

用 Web Search 代替学术数据库，效果虽不如 OpenAlex 精准，但零配置即可体验完整流程。

**步骤：**

```
1. 用户给出论点 → "多任务处理降低工作效率"
2. 拆解 → 核心断言(因果) + 关键变量(多任务→效率) + 领域(认知心理学)
3. 搜索 → web_search("multitasking productivity academic study")
4. 评估 → 扫前几条结果，挑出期刊/机构来源
5. 出报告 → APA格式引用
```

**你需要的只有**：
- 一个 AI 对话界面（任何大模型都行）
- 能联网搜索（web_search 或浏览器）
- 大概 5 分钟

> ⏩ 跳转到下方「完整工作流」看具体怎么操作。

## 配置学术 API（推荐）

接入学术数据库后，搜索精度从"网上的文章"提升到"正式发表的论文"，能拿到引用量、期刊级别、TLDR摘要。

### 你只需要两个免费 API

| API | 用途 | 注册 | 速率 |
|-----|------|------|------|
| **OpenAlex** | 主数据源，2.5亿+论文元数据 | [openalex.org](https://openalex.org/) 免费注册 | 100 req/s |
| **Semantic Scholar** | TLDR摘要、PDF链接（可选） | [semanticscholar.org/product/api](https://semanticscholar.org/product/api) 免费申请 | 1-10 req/s |

### 注册步骤（5分钟）

**OpenAlex：**
1. 打开 [openalex.org](https://openalex.org/)
2. 右上角 Sign up → 用邮箱注册
3. 登录后进入 Dashboard → 复制 API Key
4. 创建一个 `.env` 文件，写入：`OPENALEX_API_KEY=你的key`

**Semantic Scholar（可选）：**
1. 打开 [semanticscholar.org/product/api](https://semanticscholar.org/product/api)
2. 申请 API Key（免费）
3. `.env` 追加：`SEMANTIC_SCHOLAR_API_KEY=你的key`

### 依赖安装

```bash
pip install requests python-dotenv
```

    params={"search": "multitasking", "per_page": 1})

## 完整工作流

```
用户输入一段话
   ↓
Step 1 ─ 拆解论点 → 核心断言 + 断言类型 + 关键变量
   ↓
Step 2 ─ 设计搜索策略 → 构建多关键词 + 选择数据源
   ↓
Step 3 ─ 执行多轮搜索 → 直接证据 → 扩展 → 反方验证
   ↓
Step 4 ─ 信源评估 → 相关度 + 权威性 + 时效性
   ↓
Step 5 ─ 输出报告 → APA引用的结构化报告
```

### Step 1: 论点拆解

拿到用户的一句话后，提取以下要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **核心断言** | 需要验证的主张 | "多任务处理降低工作效率" |
| **断言类型** | 因果 / 相关 / 描述 / 比较 / 规范 | 因果 |
| **关键变量** | 自变量 → 因变量 | 多任务处理 → 工作效率 |
| **所属领域** | 主要学科 | 认知心理学 |
| **子断言** | 复合断言拆解 | 多任务损害效率 + 损害创造力 |

**拆解规则**：含"和/与/以及"连接的、分属不同学科的、因果链不同环节的 → 各拆成独立搜索。

### Step 2: 搜索策略设计

**数据源选择：**

| 来源 | 工具 | 使用场景 |
|------|------|---------|
| 同行评审期刊 | OpenAlex API | 主要学术证据（需配置） |
| 预印本 | arXiv | CS/AI/物理前沿 |
| 政策报告/机构出版物 | web_search | 宏观数据、政策主张 |
| 专家观点 | web_search | 辩论验证 |

**查询构建**（四层递进）：

```
第1层 ─ 直接关键词
   multitasking + productivity loss

第2层 ─ 同义词变体
   task switching + performance decline

第3层 ─ 方法论增强
   multitasking + productivity + meta-analysis

第4层 ─ 反方检查
   multitasking + no effect + productivity
```

### Step 3: 多轮搜索

| 轮次 | 目标 | 查询策略 | 期望数量 |
|------|------|---------|---------|
| 第1轮 | 直接证据 | 直接关键词搜索 | 3-5篇 |
| 第2轮 | 扩展搜索 | 同义词+方法论增强，找元分析/系统综述 | 2-4篇 |
| 第3轮 | 平衡检查 | 搜索矛盾/否定性发现 | 1-2篇 |
| 第4轮* | 权威综合 | 高引论文+权威机构报告 | 1-2篇 |

*第4轮仅在深度模式下执行。

**停止条件：**
- 轻量模式：≥5篇且≥2篇直接相关
- 深度模式：≥10篇且≥4篇直接相关
- 提前停止：第1轮已获得≥8篇高质量结果

### Step 4: 信源评估

每个来源从四个维度评估：

**1. 相关度**
| 等级 | 含义 |
|------|------|
| ★★★ 直接支持 | 同一变量、同一方向 |
| ★★☆ 间接支持 | 相关概念、相邻领域 |
| ★☆☆ 背景 | 理论框架、方法参考 |
| ⚡ 矛盾 | 相反方向、零发现 |

**2. 权威性**
| 分数 | 条件 |
|------|------|
| 高 | 顶刊 + >100引用 + 权威机构 |
| 中 | 同行评审期刊 + 20-100引用 |
| 低 | 低级别期刊 + <20引用或未经同行评审 |

**3. 时效性**
- 快速领域（CS/AI）：近3-5年
- 成熟领域（心理学/管理）：近10-15年

**4. 红旗标记**（警惕）
- 掠夺性期刊
- 样本量 < 30
- 未披露利益冲突
- 效应量大到不真实
- 作者有撤稿历史
- 行业资助且结论利好资助方

### Step 3.5: 来源分类与验证

在信源评估之后、输出报告之前，对每个进入报告的来源执行分类验证。这是防止来源类型升级和元数据虚构的关键防线。

**A. 来源类型判定**

| 类型 | 定义 | 引用格式 |
|------|------|----------|
| Type A | 同行评审期刊论文（有 DOI / PubMed ID） | APA 标准学术引用 |
| Type B | 预印本（有 arXiv / SSRN ID） | 预印本引用格式 |
| Type C | 机构正式报告（有出版编号 / ISBN） | 报告引用，注明机构+编号 |
| Type D | 网站文章 / 新闻稿 / 博文（仅有 URL） | 网页引用，附 URL + 访问日期 |
| Type E | 政府数据 / 统计（.gov 数据集） | 数据集引用格式 |

**规则：**
- Type A/B → 直接以学术引用格式写入参考文献
- Type C → 以报告引用格式写入
- Type D → 以网页引用格式写入。**如果 Type D 引用了某项研究，触发回溯协议**
- Type E → 以数据集引用格式写入

**B. 回溯协议（Traceback Protocol）**

当 Type D 来源引用了某篇研究时：
1. 从二手源中提取被引研究的线索（作者/年份/标题片段/期刊名）
2. 在学术数据库（Semantic Scholar / OpenAlex / Crossref）中搜索该研究
3. **找到** → 以一手研究的真实元数据为准，引用时标注 `（as cited in [二手源]）`
4. **未找到** → 保留 Type D 引用，标注 `[SECONDARY - primary not located]`
5. **不得**将二手源描述升级为正式学术引用

**C. 引用实体验证**

对每个 Type A 来源：
1. 在至少一个学术数据库中搜索（标题 + 第一作者 + 年份）
2. 比对：标题、作者、年份、期刊名
3. 完全匹配 → `✓`
4. 部分匹配 → `⚠`，在报告中注明差异
5. 完全未找到 → `✗`，降级为 Type D 或移除

### Step 5: 输出报告

**标准结构：**

```markdown
# 📚 证据报告：{声明原文}

## 一、声明复述
> "{原始声明}"
**断言类型**：{因果/相关/...} | **关键变量**：{自变量→因变量}

## 二、搜索概要
搜索轮次、评估来源、使用数据库、搜索时间

## 三、支持证据
### ★★★ 直接支持
**1. {作者}({年份})** — 核心发现 + 原文引用 + 证据力度 + 验证状态(✓/⚠/✗)

### ★★☆ 间接支持
**2. {作者}({年份})** — ...

## 四、反方证据
矛盾发现及解释

## 五、综合评估
整体判断、Top 3推荐来源、边界条件、建议后续搜索

## 六、参考文献（APA 7）
```

**轻量模式**（快速版）：
```
| 来源 | 年份 | 核心发现 | 支持程度 | 验证 |
|------|------|---------|---------|------|
| {Author} | {year} | 一句话 | ★★★ 直接 | ✓ |
```

## 引用格式

### APA 7（默认）

```
Journal:
Author, A. A., & Author, B. B. (Year). Title. Journal, Volume(Issue), pages. DOI
Book:
Author, A. A. (Year). Title. Publisher.
```

### GB/T 7714（中文期刊）

```
期刊论文：
作者. 题名[J]. 刊名, 年份, 卷(期): 页码.
图书：
作者. 书名[M]. 出版地: 出版社, 年份.
```

## 真实案例

> 论点：**多任务处理降低工作效率**

### 拆解结果
- **断言类型**：因果
- **关键变量**：媒体多任务处理 → 认知表现
- **领域**：认知心理学
- **子断言**：多任务损害注意力 + 损害记忆力

### 搜索摘要

| 轮次 | 查询 | 数据源 | 结果 |
|------|------|--------|------|
| 1 | media multitasking academic performance | OpenAlex | 15篇候选 |
| 2 | multitasking meta-analysis cognitive | OpenAlex | 8篇候选 |
| 3 | multitasking no effect productivity | OpenAlex | 未找到反方证据 |
| 筛选后 | — | — | **3篇精选** |

### 精选证据

| # | 论文 | 期刊 | 引用 | 核心发现 |
|---|------|------|------|---------|
| ★★★ | Madore et al. (2020). Memory failure predicted by attention lapsing and media multitasking. | *Nature* | 175 | 多任务→注意力漂移→记忆力下降，因果路径清晰 |
| ★★☆ | Van Der Schuur et al. (2018). Efficient, helpful, or distracting? | *IJETHE* | 272 | 系统综述：媒体多任务与学业表现负相关 |
| ★★☆ | Wammes et al. (2018). Disengagement during lectures. | *Computers & Education* | 116 | 课堂多任务→走神增加→测验成绩下降 |

### 结论

> 该论断有坚实的学术证据支撑。从顶刊 *Nature* 到系统综述，研究一致指向：媒体多任务通过削弱注意力持续性来损害记忆力和任务完成质量。未找到站得住脚的反方证据。

## 注意事项

### 反混编规则

- **禁止跨源合成引用**：一个来源 = 一条引用，信息来自 A 就引 A，来自 B 就引 B。
- **禁止来源类型升级**：网站文章 ≠ 期刊论文。新闻稿中的研究描述 ≠ 该研究的正式论文。
- **禁止数据点编造**：引用中所有数字（样本量、效应值、百分比）必须在原文能找到对应表述。
- **二手源标注义务**：直接来源是新闻稿/博文而非论文原文的，必须标注 `[as cited in ...]`。

### 通用规则

1. **不要编造来源** — 每个来源必须通过实际搜索找到
2. **始终搜索反方证据** — 至少一轮专门搜矛盾发现
3. **区分相关性与因果性** — 声明因果但研究只显示相关，必须明确指出
4. **标注低权威来源** — 博客/掠夺性期刊要旗帜鲜明地标出来
5. **保留原文引用** — 用原文引用，不转述概括
6. **默认 APA 7** — 用户可指定其他格式
